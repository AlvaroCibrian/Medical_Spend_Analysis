1. Objetivo de la fase

La regresión lineal múltiple construye una ecuación que estima el gasto médico (charges) a partir de varias variables a la vez, y —lo más importante para el proyecto— cuantifica cuánto pesa cada factor en dólares. Responde: «manteniendo lo demás igual, ¿cuánto sube el gasto por ser fumador, por cada año de edad, por cada punto de IMC?».

2. El modelo y cómo se ajusta

MODELO DE REGRESIÓN LINEAL MÚLTIPLE
ŷ = β₀ + β₁·age + β₂·sex + β₃·bmi + β₄·children + β₅·smoker + … + ε

ŷ es el gasto predicho; β₀ el intercepto; cada βⱼ dice cuánto cambia el gasto por una unidad de esa variable, con las demás constantes; ε es el error (lo que el modelo no explica). Los coeficientes se eligen por mínimos cuadrados ordinarios (OLS): minimizan la suma de los errores al cuadrado.

CRITERIO OLS · SOLUCIÓN MATRICIAL
min Σ (yᵢ − ŷᵢ)²   ⟶   β = ( Xᵀ X )⁻¹ Xᵀ y

La fórmula matricial da los coeficientes de golpe: X es la tabla de datos (con una columna de unos para el intercepto), y el vector de gastos. No necesitas calcularla a mano: statsmodels lo hace.

3. Código explicado (statsmodels, recomendado)

Se recomienda statsmodels porque su summary() entrega R², coeficientes y p-values juntos — justo lo que pide el profesor.

import statsmodels.api as sm

# 1) Separar variables independientes (X) y dependiente (y)
X = df_numerico.drop(columns="charges")     # age, sex, bmi, children, smoker, region_*
y = df_numerico["charges"]

# 2) Agregar la columna de intercepto (la constante β0)
X = sm.add_constant(X)

# 3) Ajustar el modelo OLS y ver el resumen
modelo = sm.OLS(y, X).fit()
print(modelo.summary())

Línea por línea: drop(columns="charges") deja solo las X; add_constant agrega el término β₀ (sin él, la recta pasaría forzada por el origen); OLS(y, X).fit() resuelve los mínimos cuadrados; summary() imprime la tabla que hay que interpretar y capturar en pantalla.

Alternativa scikit-learn (para predecir y medir error en datos nuevos): LinearRegression().fit(X_train, y_train) + r2_score, mean_squared_error. Lo ideal es usar ambas: statsmodels para interpretar, sklearn con train_test_split para evaluar honestamente en datos no vistos.

4. Cómo leer el resumen del modelo (lo que hay que capturar)

4.1 R² — ¿qué tan bueno es el modelo?

COEFICIENTE DE DETERMINACIÓN
R² = 1 − [ Σ (yᵢ − ŷᵢ)² ] / [ Σ (yᵢ − ȳ)² ]

Proporción de la variación del gasto que el modelo explica (0 a 1). Usa el R² ajustado para comparar modelos, porque penaliza variables inútiles.

Resultado esperado (verificado con literatura): este modelo alcanza R² ≈ 0.75–0.76 — explica ~¾ de la variabilidad del gasto (Lantz, 2019; estudios replicados sobre el mismo dataset).

4.2 Coeficientes — ¿cuánto pesa cada variable?

Cada coeficiente está en dólares. Ejemplo de lectura: si el coeficiente de age es ~256, cada año adicional se asocia con ~$256 más de gasto, con lo demás constante.

Coeficiente estrella (verificado): el de smoker ronda +$23 800 (estudios reales reportan 23 848 y 23 811). Es decir: a igualdad de edad, IMC, etc., ser fumador se asocia con ~$23 800 más de gasto anual. Es, con enorme diferencia, el mayor efecto.

4.3 P-values — ¿qué variables son significativas?

El p-value es la probabilidad de ver este efecto por puro azar si la variable en realidad no influyera. Regla estándar: p < 0.05 ⟶ la variable es estadísticamente significativa. Ojo: significativo no es lo mismo que «grande»; solo dice que el efecto probablemente no es casualidad.

Esperado: age, bmi y smoker salen muy significativas (p muy pequeña); sex y algunas region_* suelen no ser significativas (p > 0.05).
Malinterpretación común. Un p-value alto (>0.05) NO prueba que la variable «no sirve»; solo que en estos datos no hay evidencia suficiente de su efecto. No lo redactes como «no influye», sino como «no resultó estadísticamente significativa».

5. Supuestos y una mejora recomendada

OLS es óptimo (BLUE) si se cumplen los supuestos de Gauss-Markov: linealidad, independencia, no multicolinealidad perfecta, exogeneidad y homocedasticidad (varianza del error constante).

Mejora que suma puntos: por el sesgo de charges, es casi seguro que se viole la homocedasticidad. Ajusta un segundo modelo con y = np.log(df_numerico["charges"]) y compara el R². La transformación log estabiliza la varianza y suele mejorar el ajuste (Feng et al., 2014). Interpretar coeficientes en escala log cambia (pasan a ser ~% de cambio), así que explícalo.
